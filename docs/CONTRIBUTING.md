# Things you can contribute

## Translations
If you want to add translations, you may want to make them for the original instead of this port, most translations are automatically downloaded from there anyway.

Should you want to translate parts where copying the translation of the original is impossible, you can do that in the language providers located in `fabric/src/main/java/net/fawnoculus/ntm/fabric/client/datagen/lang/`

Please also keep in mind that all translations that have been changed should be made available in their original form in the built-in legacy resource pack, and changes documented in [Changes.md](Changes.md)

## Documentation
Since the [NTM Wiki](https://nucleartech.wiki/) is only made for the original version of NTM it may be beneficial to keep a record of all things that where changed,
see [Changes from Original](./Changes.md).

Admittedly, [That changes document](./Changes.md) is not the best in terms of having good formating or being up to date on the current state of the port.
If you have any fixes to that, I would be grateful.

If you want to make documentation for features from the original mod I would recommend contributing to the [NTM Wiki](https://nucleartech.wiki/) instead of doing work here.

If instead you wish to document code of this mod for future contributors (or addon developers), you can add Javadocs to everything that doesn't have them yet.

# Some Guidelines to follow

## For Code Changes
1. If you make any code changes, please make sure that they actually work before putting them in a Pull Request.
   (just stating the game and seeing if your changes broke whatever you changed allready catches plenty of bug)
2. Please try to communicate your proposals for big changes before just working on them yourself,
   lots of issues can be solved significantly quicker with just a little communication!
